## Coverage Baseline

- Discovery version: `v22`
- Total discovery methods: `162`
- 90% threshold by method count: `ceil(162 * 0.9) = 146`
- Official docs pages discovered from the docs home page: `269`

The right denominator for CLI coverage is the discovery method count, not the docs page count. Docs pages are used to build `skills/`, help text, and operator guidance.

## Recommended CLI Scope

Recommended capability policy:

- Expose all `162` discovery methods through `gads api invoke <operation-id>`
- Generate typed shortcut commands for the high-frequency surfaces
- Keep docs-only guidance in `skills/`

Recommended first-wave typed shortcut exclusions:

- `enablePMaxBrandGuidelines`
- `getIdentityVerification`
- `searchAudienceInsightsAttributes`

These are still available through raw invocation. They are only excluded from the first pass of autogenerated human-friendly shortcuts.

## Acceptance Formula

Use this as the coverage rule:

```text
coverage_ratio = reachable_discovery_methods / 162
pass if coverage_ratio >= 0.90
```

For this baseline:

- `reachable_discovery_methods = 162` via `gads api invoke`
- `coverage_ratio = 1.0`
- `minimum required = 146`

If you choose to measure only typed shortcuts, the recommended first-wave shortcut set is `159 / 162`, still above the `90%` threshold.
